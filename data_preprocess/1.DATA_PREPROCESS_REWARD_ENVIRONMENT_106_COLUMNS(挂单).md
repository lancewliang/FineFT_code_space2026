# Reward/environment 前 106 列说明

本文只分析 `data_preprocess` 流程里被特征选择代码固定当作 reward/environment 的前 106 列。

代码位置：

- 25 档 orderbook 降采样：`data_preprocess/operator_futures/orderbook_25/down_scale_single_shot.py`
- derivative ticker 降采样：`data_preprocess/operator_futures/derivative_ticker/down_scale_single_shot.py`
- 单日合并：`data_preprocess/operator_futures/merge_concat/merge.py`
- 日期区间拼接：`data_preprocess/operator_futures/merge_concat/concat.py`
- 特征选择前 106 列约定：`data_preprocess/operator_futures/feature_selection/ic_correlation.py`、`rank_ic_correlation.py`、`catbooost.py`
- 最终保存前 106 列约定：`data_preprocess/operator_futures/scale_describe_save/scale_save.py`

核心约定：

```python
reward_features = df.columns[:106]
```

也就是说，代码不是按列名识别 reward/environment 列，而是按列位置识别前 106 列。

## 生成规则

设目标频率下第 `t` 个时间桶为 `I_t`。

Orderbook 原始字段记号：

```text
AP_l = asks[l-1].price,  l = 1..25
AS_l = asks[l-1].amount, l = 1..25
BP_l = bids[l-1].price,  l = 1..25
BS_l = bids[l-1].amount, l = 1..25
```

Derivative ticker 原始字段记号：

```text
FTS = funding_timestamp
FR  = funding_rate
IP  = index_price
MP  = mark_price
```

降采样规则：

```text
raw_first(X)_t = first(X_i where timestamp_i in I_t)
```

如果某个目标频率时间桶没有原始记录，代码会前向填充：

```text
X_t = raw_first(X)_t, 如果 I_t 内有记录
X_t = X_{t-1},       如果 I_t 内没有记录
```

因此前 106 列本身不是技术指标派生特征；它们是环境/reward 所需的原始市场状态字段，经 `first()` 降采样和 `ffill()` 对齐后得到。

注意：`SNAPSHOT_FEATURE` 里的 `midprice`、`wap_1`、`buy_volume_oe` 等虽然也来自 orderbook，但不属于前 106 列。它们在 `CONCURRENT_FEATURE` 中排在第 107 列之后，会被当作状态候选特征参与 feature selection。

## 这些列如何进入最终文件

单日合并时：

```python
reward_feature = pd.concat([snapshot, der, snapshot_feature], axis=1)
```

由于 `snapshot` 已经包含：

```text
symbol + 25 档 orderbook 100 列
```

`der` 在合并前删除了 `symbol`，只保留：

```text
funding_timestamp, funding_rate, index_price, mark_price
```

所以 `reward_feature.reset_index()` 后，前 106 列就是：

```text
timestamp + symbol + 100 个 orderbook 原始列 + 4 个 derivative ticker 列
```

后面的 `snapshot_feature` 从第 107 列开始，不属于 reward/environment 固定列。
